### 2. Chrome DevTools MCP Compatibility Fix

**Problem:** Chrome DevTools MCP fails silently with MiMo V2.5 model but works with MiniMax M3 Free.

**Root Cause:** MiMo V2.5's Xiaomi API has strict limitations:
- Only supports single-round tool calling (Chrome DevTools needs multiple rounds)
- Rejects list-type tool message content (screenshots return as lists)
- Has strict schema validation that rejects `anyOf`/`nullable` patterns

**Solution Implemented:**
1. Updated `opencode.jsonc`:
   - Added `--autoConnect` and `--channel=stable` flags to Chrome DevTools MCP for more stable connections
2. Updated `AGENTS.md`:
   - Added "Model Switching Strategy" section explaining when to use MiMo vs MiniMax
   - Updated step 7 (Chrome DevTools testing) with MiMo limitation warning and workaround instructions
   - Clear guidance: Use MiniMax for Chrome DevTools testing, MiMo for general coding

**Key Insight:** This is a Xiaomi API limitation, not an OpenCode or project issue. The ChromeDevTools MCP team has tried to fix it (PR #1073 removed `.nullable()` from emulation tools), but MiMo's fundamental API limitations remain.

**Workaround:** Switch to MiniMax M3 Free (`opencode-zen/minimax-m3-free`) or Blackbox AI MiniMax (`blackboxai/minimax/minimax-free`) when testing tools with Chrome DevTools MCP.

### 3. Added Duplicate Check Step to Tool-Building Workflow

**Problem:** No formal step to check if a tool already exists before building it. Risk of building duplicates or tools with overlapping functionality.

**Solution:** Added Step 0 (Duplicate Check) as a BLOCKING first step to both:
1. `AGENTS.md` — Workflow section (step 0 before step 1)
2. `.opencode/skills/tool-builder/SKILL.md` — "Before any creation or update" section

**What it checks:**
- `src/data/tools.json` by name, category, keywords
- `src/tools/` directory for similar functionality
- `toolsList.json` for registered tools

**Decision logic:**
- Same function, same name → STOP (already exists)
- Same function, different name → STOP (suggest extending existing)
- Partial overlap → Note it, propose differentiation, get user confirmation
- No overlap → Proceed
