### Image-tool factory design
Pattern A scaffold (file upload + HTML + state + download) extracted into
`src/tools/image/image-tool-factory.js`:
- `createImageTool({ container, toolId, multiple, optionsHTML, optionsCSS, renderPreview, processForDownload, getFilename, getFormat, getQuality, successMessage, onImageLoaded, processingMessage })`
- Returns `{ state, elements, bindOptionChange, renderPreview }`
- Handles file upload (single or multi), state.originalImage vs state.images[],
  show-on-upload logic, download button with spinner + format/quality + toast
- bindOptionChange({ rangeId, valueId, formatValue?, onChange? }) wires the
  (range -> val -> renderPreview) pattern

Tools now provide only:
- optionsHTML (form controls)
- optionsCSS (scoped styles)
- renderPreview + processForDownload (per-tool drawing)
- getFilename/getFormat/getQuality (per-download resolvers)
- Custom listeners for tool-specific widgets (style buttons, mode buttons, color pickers)

Tools refactored: add-border-image, blur-background, round-image-cropper, collage-maker.
Net: 4 tools went from 1276 -> 1046 (-230 lines) + 195-line factory.
New image tools become ~50-line stubs.

### Pre-existing bugs caught during refactors
- `calorie-estimator.js`: macros (protein + carbs + fat) didn't sum to TDEE (rounding inconsistency). Fixed by extracting `computeMacros(tdee, weight)` (1.6g/kg protein + 50/50 split of remaining calories).
- `qr-scanner.js`: `html5qrcode.scanImage(canvas, {verbose: true})` — non-existent API (only `scanFile`/`scanFileV2` exist). Also leaked `URL.createObjectURL` blob URL (never revoked). Removed entire fallback.
- `barcode-scanner.js`: `result.result.format?.formatName` threw TypeError on `scanFile()` string returns. Switched to `scanFileV2()` returning `Html5QrcodeResult`. Now actually shows decoded barcode format.
- `scanner-factory.js`: `window.open(url, '_blank')` was tabnabbing-vulnerable (user scans attacker-controlled QR). Added `'noopener,noreferrer'`.
- 4 of 6 lookup tools had innerHTML XSS via API responses (dictionary, thesaurus, holiday-calendar, book-lookup). All fixed via `escapeHtml()`.

### TDD pattern for browser-library bugs
Mock at the prototype level: `vi.spyOn(Html5Qrcode.prototype, 'scanFileV2')`.
For window.open: `vi.stubGlobal('open', vi.fn())` (vi.spyOn leaks call counts across tests in this codebase).
For DragEvent in jsdom: use `new Event('drop', { bubbles: true })` + `Object.defineProperty(dropEvent, 'dataTransfer', { value: { files: [file] } })`.
For image loading: mock `loadImageFromFile` (jsdom doesn't decode images).
For canvas.toBlob: mock `canvasToBlob` (jsdom canvas is a stub).
